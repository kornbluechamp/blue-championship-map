Review workflow

Load the published data or import a JSON file. Click any point or route to edit its name, type/status, nearest area, and notes. Enable point dragging to correct a marker location. Name and category changes can be republished directly. Moving points, deleting paths, or changing route access requires the routing network to be rebuilt.

Publishing step

The public export downloads a file named verified-data.json. Upload that file into the root of the GitHub repository and replace the existing file. After GitHub Pages deploys, all spectators and volunteers using the main link will see the published points and path lines automatically.
